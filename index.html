¿Què son las VIEWS en MYSQL?

Las vistas son una forma de simplificar y optimizar el acceso a los datos de una o más tablas. Es una tabla virtual que se almacena en la base de datos como un objeto. Las vistas tienen la misma estructura de filas y columnas que una tabla normal. Se guardan en la base de datos como consultas con nombre y se pueden utilizar para guardar consultas completas que se utilizan con frecuencia.

¿Que son los procedimientos almacenados?

Es un conjunto de instrucciones que se pueden llamar desde otras consultas o desde otros procedimientos almacenados. Un procedimiento puede tomar argumentos de entrada y mostrar valores como resultados. Debes nombrar y almacenar un procedimiento en un conjunto de datos de BigQuery. Un procedimiento almacenado puede acceder a los datos o modificarlos en varios conjuntos de datos de varios usuarios.

¿Cuál es la diferencia entre SQL y NoSQL?

SQL

SQL es una base de datos relacional que organiza los datos en tablas con filas y columnas. SQL significa Lenguaje de consulta estructurado, que es el lenguaje estándar utilizado para consultar y manipular datos en una base de datos relacional.
*Es estricto en su interacción, por lo tanto, el desarrollador deberá de conocer a la perfección como manipular el lenguaje de manera exacta y única.
*En SQL se emplea escalabilidad horizontal (fragmentación), donde dividen las tablas en particiones más pequeñas y las distribuyen en varios servidores, haciendo que se requiera mas potencia de hardware.
*SQL posee una mayor comunidad activa, ya que este lleva activa los años 70s, por lo que se refleja en mayor soporte.

NoSQL

Una base de datos NoSQL es un base de datos no relacional que almacena datos en un formato distinto de filas y columnas. Estos se almacenan con objetos, tal como se realiza en JavaScript, siendo más flexible a la hora de realizar la base de datos.
*NoSQL emplea documentos JSON para el almacenamiento de datos, a diferencia de SQL que si utiliza un lenguaje de consulta.
*Se pueden manejar diversos métodos de interactividad, por lo tanto es mucho mas flexible y creativo a comparación de SQL.
*NoSQL usa la escalabilidad vertical, generando mayor rendimiento al agregar recursos a un servidor, lo que le brinda más flexibilidad. La escalabilidad vertical es popular para las aplicaciones basadas en la nube, y por lo tanto, más eficiente y menos costosa.
*NoSQL a pesar de no tener una comunidad tan grande como la tiene SQL, se ha ido implementando poco a poco, yendo en crecimiento en la industria.

¿Que es MongoDB

MongoDB es una base de datos NoSQL de código abierto. NoSQL significa que la base de datos no utiliza tablas relacionales como una base de datos SQL tradicional.
En MongoDB se almacena los datos en objetos similares a JavaScript por medio de objetos, donde se refleja cada uno de los elementos que componen la base de datos realizada.
A diferencia de SQL, no hay límites estructurales para almacenar datos en MongoDB. Puedes almacenar lo que quieras donde quieras, por lo tanto, puedes generar objetos con los datos de manera natural y no estricta.
